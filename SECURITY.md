# Security policy

## Supported version

Security fixes target the latest commit on `main`.

## Reporting a vulnerability

Do not open a public issue for a vulnerability. Use the repository's **Security** tab and choose **Report a vulnerability** to open a private GitHub security advisory.

Do not include real credentials or personal data in a report. If a secret is exposed, revoke or rotate it at the provider first; removing it from the latest commit is not sufficient because Git history retains old blobs.
